# 版本变更记录

本文档用于记录每个版本的用户可见变化。每次发版前必须更新本文件，并同步检查 `docs/版本迭代记录规范.md`。

## 未发布
- 在这里记录当前迭代中的功能、修复和调整。
- 补充标准开发流程文档，并将 GitHub 版本管理流程接入交接与维护文档。
- 为 Git 初始化、打标签、标准发版脚本补充 `safe.directory` 自动处理，降低跨账户/跨终端环境下的仓库权限报错。
- 新增版本说明自动导出脚本，并将 `release-notes.md` 纳入标准交付物与 GitHub Release 流程。

## 1.0.2 - 2026-05-17

- 补齐 AI 设置入口与统计页 AI 复盘入口。
- 完善 Windows 卸载体验：卸载时可选择是否清理用户数据。
- 发布包增加“测试数据/用户数据排除”与发布前自动检查，避免把本机数据带给用户。
- 统一 `02-user-software` 和 `03-send-package` 的交付结构。
- 新增 GitHub 版本管理骨架：`.github` 模板、仓库初始化脚本、发布标签脚本、中文流程文档。

## 1.0.1 - 2026-05-17

- 修复桌面端在部分用户环境中错误指向局域网 API 地址（如 `10.x.x.x:18765`）导致“无法连接服务/请求超时”的问题。
- 优化桌面端 API 地址解析策略：优先同源，异常场景回退到 `127.0.0.1`。
- 重写全局异常页，修复中文乱码。
- 异常页新增“重新加载”和“复制诊断”按钮，便于用户自助排障和远程支持。
- `Stats` 页新增 AI 总结本地历史记录（查看、复制、导出、清空）。

## 1.0.0 - 2026-05-15

- 完成 Windows-first 本地运行链路。
- 支持前后端启动脚本、本地 SQLite 存储与 Windows 桌面打包。
- 支持安装包导出、用户软件目录整理与发送压缩包生成。
- 补充开发交接文档、开发维护文档和本地清理脚本。
