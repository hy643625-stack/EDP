# 图

## 学习目标
- 能区分图的表示方法（邻接矩阵、邻接表）及其适用场景
- 能用 DFS 和 BFS 遍历图并解决连通分量、拓扑排序等基本问题
- 能实现并理解 Dijkstra 最短路径算法
- 能解释最小生成树（Prim / Kruskal）的原理与实现

## 知识地图
```
图论基础
├── 图的表示
│   ├── 邻接矩阵 O(V²) 空间 / O(1) 判边
│   └── 邻接表 O(V+E) 空间 / O(degree) 扫描邻居
├── 图的遍历
│   ├── DFS（栈/递归）
│   │   ├── 连通分量、环检测、拓扑排序
│   │   └── 强连通分量（Tarjan / Kosaraju）
│   └── BFS（队列）
│       ├── 无权最短路、层次遍历
│       └── 二分图检测
├── 最短路径
│   ├── Dijkstra（非负权，贪心 + 优先队列）O(E log V)
│   ├── Bellman-Ford（含负权）O(VE)
│   └── Floyd-Warshall（全源）O(V³)
└── 最小生成树
    ├── Prim（类似 Dijkstra，按边权）O(E log V)
    └── Kruskal（排序 + 并查集）O(E log E)
```

## 核心讲解

### 图的表示与选择
| 维度 | 邻接矩阵 | 邻接表 |
|------|---------|--------|
| 空间 | O(V²) | O(V+E) |
| 判断边(u,v) | O(1) | O(degree(u)) |
| 遍历邻居 | O(V) | O(degree) |
| 适合 | 稠密图、Floyd-Warshall | 稀疏图、大多数算法 |

### DFS vs BFS 的本质区别
- **DFS**：深度优先，适合"是否存在"、"全部枚举"类问题
- **BFS**：广度优先，适合"最优"、"最短"、"最近"类问题

两者时间和空间复杂度均为 O(V+E)，区别在于使用的辅助结构（栈 vs 队列）和节点被访问的顺序。

### Dijkstra 算法
Dijkstra 的核心思路：贪心地选择"当前已知最短距离最小"的未确定节点，松弛其所有邻居。

**为什么不能有负权边？**
因为贪心假设了"已经确定最短距离的节点不会被后来的路径更新"。负权边可能让一条经过更多节点但总权更小的路径出现，打破贪心假设。

### 最小生成树
- **Prim**：从一个点开始生长，每次选最小边连接树内外
- **Kruskal**：从最小边开始，如果不形成环就加入

关键区别：Prim 维护一棵不断生长的树；Kruskal 维护一个森林，逐步合并。

## 易错点
- 无向图遍历时未标记父节点导致"回边"被误判为环
- Dijkstra 中使用大顶堆而非小顶堆
- BFS 用栈（先进后出）而非队列（先进先出）
- Kruskal 未做路径压缩导致并查集退化
