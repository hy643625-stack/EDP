# 复杂度与抽象数据类型

## 学习目标
- 能区分时间复杂度和空间复杂度
- 能用大 O 表示常见循环结构的复杂度
- 能解释抽象数据类型（ADT）与具体实现的区别
- 能分析简单递归函数的时间复杂度

## 知识地图
```
复杂度分析
├── 时间复杂度（大 O、大 Ω、大 Θ）
├── 空间复杂度（原地算法、辅助空间）
├── 渐进分析（最好/最坏/平均情况）
└── 递归分析（递推式、主定理）

抽象数据类型（ADT）
├── 概念：接口与实现分离
├── 常见 ADT：List、Stack、Queue、Set、Map
└── 数据结构：ADT 的具体实现方式
```

## 核心讲解

### 为什么要分析复杂度？
算法效率直接决定程序能否在合理时间内处理实际规模的数据。复杂度分析让我们在编写代码前就能预估算法的扩展性。

### 时间复杂度
时间复杂度描述算法运行时间随输入规模 n 增长的变化趋势。我们关注的是增长量级，而非具体秒数。

| 常见复杂度 | 名称 | n=10⁶ 时的参考操作数 |
|-----------|------|---------------------|
| O(1) | 常数 | 1 |
| O(log n) | 对数 | ~20 |
| O(n) | 线性 | 10⁶ |
| O(n log n) | 线性对数 | ~2×10⁷ |
| O(n²) | 平方 | 10¹² |
| O(2ⁿ) | 指数 | 不可行 |

### 空间复杂度
衡量算法运行所需的额外存储空间。递归程序的调用栈也计入空间复杂度。

### 抽象数据类型（ADT）
ADT 是面向用户的逻辑规范，只定义"做什么"而不规定"怎么做"。例如：
- 栈（Stack）ADT：push、pop、peek 操作
- 实现可以是数组，也可以是链表

这种分离带来了模块化和可替换性的优势。

### 递归与主定理
递归是描述和解决问题的重要工具。主定理（Master Theorem）是分析分治算法复杂度的利器：

对于 T(n) = aT(n/b) + f(n)：
- 若 f(n) = O(n^{log_b a - ε})，则 T(n) = Θ(n^{log_b a})
- 若 f(n) = Θ(n^{log_b a})，则 T(n) = Θ(n^{log_b a} log n)
- 若 f(n) = Ω(n^{log_b a + ε})，则 T(n) = Θ(f(n))

## 典型例子
1. **线性搜索**：O(n) 时间，O(1) 空间
2. **冒泡排序**：O(n²) 时间，O(1) 空间
3. **归并排序**：O(n log n) 时间，O(n) 空间
4. **斐波那契递归**：O(2ⁿ) 时间（无记忆化），O(n) 空间（调用栈）

## 易错点
- 误把 O(2n) 当作不同的复杂度 —— 大 O 忽略常数因子
- 认为 O(1) 一定比 O(n) 快 —— 当 n 很小时常数因子可能主导
- 分析递归时只考虑深度而不考虑每层的工作量
- 把 ADT 和数据结构混为一谈

## 自测问题
1. 一个算法遍历数组两次，复杂度是 O(2n) 还是 O(n)？为什么？
2. 递归深度为 n 且每层只做常数工作的算法，空间复杂度是多少？
3. 请列举三个 ADT，并说明各自的核心操作。

## 与后续模块的关系
本章的复杂度分析方法是所有后续模块的基础工具。后续学习线性表、树、图等数据结构时，需要持续运用大 O 分析各操作的时间/空间效率。ADT 的概念将在栈、队列等模块中反复实践。
