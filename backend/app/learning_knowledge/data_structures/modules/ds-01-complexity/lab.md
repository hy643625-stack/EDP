# 实验：复杂度观察

## 任务目标
编写程序对比不同算法在同一输入规模下的实际运行时间，验证复杂度理论分析。

## 输入输出
- 输入：不同的输入规模 n（如 10², 10³, 10⁴, 10⁵）
- 输出：表格记录各算法在不同 n 下的运行时间（ms），绘制增长趋势图

## 实现步骤
1. 实现以下算法（可调用标准库，但需手写核心逻辑）：
   - 线性搜索（Linear Search）
   - 冒泡排序（Bubble Sort）
   - 内置排序（sorted / list.sort，作为 O(n log n) 参照）
2. 对每个 n 值生成随机整数数组
3. 使用 `time.perf_counter()` 测量每个算法的执行时间
4. 对每组实验重复 3 次，取平均值
5. 将结果输出为 Markdown 表格，或用 matplotlib 绘制双对数坐标图

## 提交物
1. Python 脚本 `complexity_observer.py`
2. 实验结果表格（嵌入 Markdown 报告）
3. 简短分析：实验结果是否与理论复杂度一致？如不一致，可能原因是什么？

## 反思问题
1. 为什么 n=10² 时冒泡排序可能比内置排序"感觉"更快？
2. 如果 n=10⁷，线性搜索和内置排序哪个更可行？为什么？
3. 实验中还有哪些因素可能影响运行时间测量（CPU 缓存、GC、OS 调度）？
