# 实验：哈希表实现与性能测试

## 任务目标
从零实现链地址法哈希表，并与 Python dict 进行性能对比。

## 输入输出
- 输入：随机生成的键值对集合
- 输出：自实现哈希表 vs dict 的插入、查找、删除耗时对比

## 实现步骤
1. 实现 `MyHashMap` 类：
   - 哈希函数：`hash(key) % capacity`（或自定义简单哈希）
   - 链地址法：每个槽是 (key, value) 元组列表
   - `put(key, value)`：插入或更新
   - `get(key)`：查找，未找到返回 None 或抛 KeyError
   - `delete(key)`：删除
   - 负载因子 > 2 时自动扩容（翻倍 + rehash）
2. 编写性能测试：
   - 插入 n=1000, 10000, 100000 个键值对
   - 随机查找 10000 次
   - 随机删除 1000 次
   - 对比 Python 内置 dict
3. 用不同负载因子触发场景测试扩容前后的性能变化
4. 输出对比表格和简要分析

## 提交物
1. `my_hashmap.py` — 哈希表实现
2. `hashmap_benchmark.py` — 性能对比脚本
3. 实验结果表格和分析

3. 哈希表 vs 排序 + 二分查找，在"一次预处理 + 多次查询"场景下应如何选择？
