# 实验：排序算法对决

## 任务目标
实现 5 种排序算法并在多种数据分布下进行性能对决，理解理论与实践的差异。

## 输入输出
- 输入：5 种不同分布的测试数据（随机、升序、降序、大量重复、小规模）
- 输出：各算法在不同分布和规模下的运行时间对比矩阵

## 实现步骤
1. 实现 5 种排序函数（允许使用简单版本）：
   - 冒泡排序（含提前终止优化）
   - 插入排序
   - 归并排序
   - 快速排序（随机 pivot + 三数取中两个版本）
   - 堆排序
2. 添加 Python 内置 `sorted()`（Timsort）作为基准
3. 生成测试数据：
   - n = 100, 1000, 10000, 50000
   - 分布：随机、已升序、已降序、所有元素相同
4. 对每个 (n, 分布) 组合运行所有算法，测量耗时
5. 输出 Markdown 格式的性能对比表和分析

## 提交物
1. `sorting_algorithms.py` — 5 种排序算法实现
2. `sorting_benchmark.py` — 性能测试框架
3. 性能对比结果和分析（Markdown 报告）

## 反思问题
1. 在什么数据分布下快速排序比 Timsort 更快？
2. 对小数组（n<50），哪种算法表现最好？为什么 Python 的 Timsort 要在小分区上使用插入排序？
3. 排序算法的常数因子在什么场景下比渐进复杂度更重要？
