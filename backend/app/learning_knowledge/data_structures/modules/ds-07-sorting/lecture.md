# 排序算法

## 学习目标
- 能实现并对比 6 种经典排序算法的复杂度和稳定性
- 能分析不同排序算法在有序度、数据规模和内存约束下的适用性
- 能解释归并排序的分治思想和快速排序的划分策略
- 能在合适场景选用桶排序、计数排序、基数排序等非比较排序

## 知识地图
```
排序算法
├── 简单排序 O(n²)
│   ├── 冒泡排序（稳定）
│   ├── 插入排序（稳定、近乎有序时接近 O(n)）
│   └── 选择排序（不稳定）
├── 高效排序 O(n log n)
│   ├── 归并排序（稳定、需要 O(n) 辅助空间）
│   ├── 快速排序（不稳定、平均最优的常数因子）
│   └── 堆排序（不稳定、O(1) 辅助空间、就地排序）
├── 非比较排序 O(n + k)
│   ├── 计数排序（整数、范围小）
│   ├── 桶排序（均匀分布）
│   └── 基数排序（按位排序）
└── 混合排序
    └── Timsort（Python 内置，归并 + 插入，稳定）
```

## 核心讲解

### 排序算法全景对比

| 算法 | 平均 | 最坏 | 空间 | 稳定 | 适用场景 |
|------|------|------|------|------|---------|
| 冒泡 | O(n²) | O(n²) | O(1) | ✓ | 教学演示 |
| 选择 | O(n²) | O(n²) | O(1) | ✗ | 写操作昂贵 |
| 插入 | O(n²) | O(n²) | O(1) | ✓ | 小规模、近乎有序 |
| 归并 | O(n log n) | O(n log n) | O(n) | ✓ | 外部排序、稳定性需求 |
| 快排 | O(n log n) | O(n²) | O(log n) | ✗ | 通用场景、常数因子小 |
| 堆排 | O(n log n) | O(n log n) | O(1) | ✗ | 内存极度受限 |

### 为什么快排通常比归并排序快？
1. **常数因子小**：快排的比较和交换在循环内部，归并有额外的数组复制操作
2. **缓存友好**：快排在原数组上操作，归并需要使用辅助数组
3. **尽管最坏是 O(n²)**，但随机化或三数取中使实际表现稳定

### 归并排序的分治精髓
```
排序 [5, 2, 4, 6, 1, 3]
├── 分：[5, 2, 4] [6, 1, 3]
│   ├── 再分：[5] [2, 4]  [6] [1, 3]
│   └── 再分：[2] [4]  [1] [3]
└── 合并：[2, 4] → [2, 4, 5]  [1, 3] → [1, 3, 6]
    → 合并：[1, 2, 3, 4, 5, 6]
```

## 典型例子
1. **求数组的逆序对数量**：归并排序改造，O(n log n)
2. **Top-K 问题**：堆排序或快速选择，O(n) 平均
3. **颜色分类**（Dutch National Flag）：三路快排思想
4. **合并 K 个有序链表**：归并思想 + 堆优化

## 易错点
- 快排 pivot 选择不当导致无限递归（如所有元素相等时）
- 把非比较排序 O(n+k) 的 k 忽略，实际上数据范围可能远大于 n
- 不理解稳定性应用场景（如按多个关键字排序时）

## 与后续模块的关系
排序是算法设计的经典案例，分治、贪心、动态规划等策略在排序算法中都有体现。归并思想是外部排序和大数据处理的基础。
