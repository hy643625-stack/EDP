# 栈与队列

## 学习目标
- 能解释栈 LIFO 和队列 FIFO 的特性
- 能用数组或链表实现栈和队列
- 能用栈处理括号匹配和表达式求值问题
- 能解释队列在 BFS 中的作用

## 知识地图
```
受限线性结构
├── 栈 Stack (LIFO)
│   ├── push / pop / peek
│   ├── 数组实现（Python list）
│   ├── 链表实现
│   └── 应用：括号匹配、表达式求值、DFS 非递归
├── 队列 Queue (FIFO)
│   ├── enqueue / dequeue / front
│   ├── 循环队列（环形缓冲区）
│   ├── 链表实现（带头尾指针）
│   └── 应用：BFS、任务调度、消息队列
└── 双端队列 Deque
    ├── appendleft / popleft / append / pop
    └── 应用：滑动窗口、回文检查
```

## 核心讲解

### 栈
栈是只能在表尾（栈顶）操作的线性表。类比现实中的一摞盘子：你只能拿最上面那个。

**应用场景**：
1. **函数调用栈**：每次函数调用压入栈帧，返回时弹出
2. **括号匹配**：遇到左括号入栈，右括号与栈顶匹配后出栈
3. **表达式求值**：中缀 → 后缀 → 计算
4. **DFS 非递归实现**：手动维护栈替代递归

### 队列
队列只允许在队尾添加、队首删除。类比排队：先来的先服务。

**循环队列优化**：
普通数组实现中，出队后前方空间闲置。循环队列通过取模 `(index + 1) % capacity` 让队尾可以"绕回"数组开头。

判断条件：
- 队空：`front == rear`
- 队满：`(rear + 1) % capacity == front`

### 表达式求值算法
1. **中缀转后缀（调度场算法）**：
   - 操作数直接输出
   - 运算符与栈顶比较优先级；栈顶优先级高则弹出栈顶并输出
   - 左括号入栈，右括号弹出至左括号
2. **计算后缀表达式**：
   - 操作数入栈
   - 遇到运算符则弹出两个操作数，计算后结果入栈

## 典型例子
1. **括号匹配**：`"({[]})"` → True, `"({[})"` → False
2. **逆波兰计算**：`["2","1","+","3","*"]` → 9
3. **用栈实现队列**：均摊 O(1) 的经典问题
4. **滑动窗口最大值**：双端队列维护窗口内的递减序列

## 易错点
- Python list 的 pop(0) 是 O(n)，不应直接用来实现队列
- 循环队列中区分空/满需要留一个空位或用计数器
- 中缀转后缀时，左括号入栈后优先级最低（特殊处理）
