# 综合实验与课程项目

## 学习目标
- 能综合运用多种数据结构解决实际工程问题
- 能根据问题特征选择最优的数据结构和算法组合
- 能对已有代码进行复杂度分析和性能优化
- 能独立完成一个含设计文档的小型课程项目

## 核心讲解

### 数据结构选型决策框架
遇到一个问题时，按以下顺序思考：

1. **分析操作模式**：
   - 哪些操作是频繁的？（查找、插入、删除、遍历）
   - 哪些操作是偶尔的？
2. **评估数据特征**：
   - 数据量多大？内存是否受限？
   - 数据是否有序？是否有重复？
   - 键的类型？（整数、字符串、自定义对象）
3. **列出候选结构**：
   - 按操作复杂度表筛选 2-3 个候选
4. **权衡取舍**：
   - 时间复杂度 vs 空间复杂度
   - 实现复杂度 vs 性能提升
   - 稳定性需求

### 综合实验：贪吃蛇的数据结构分析
```
蛇身：双端队列（deque）
  ├── 吃到食物：appendleft 新头部 O(1)
  ├── 正常移动：appendleft 新头部 + pop 尾 O(1)
  └── 错误做法：list.insert(0) O(n)

碰撞检测：集合（set）
  ├── 判断 (x, y) 是否在蛇身中 O(1)
  ├── 判断是否撞墙 O(1)
  └── 错误做法：list 遍历 O(n)
```

### 错题分析与学习方法
有效的错题复盘包含以下要素：
- **错误分类**：概念混淆 / 边界条件 / 复杂度误判 / 代码实现
- **根本原因**：不只是"粗心"，而是哪个认知模型出了问题
- **正确思路**：写出标准的解题路径
- **举一反三**：这个错误模式还可能出现在哪些场景？

## 典型例子
1. **设计排行榜系统**：哈希表存分数 + 堆取 Top-K
2. **LRU 缓存完整实现**：哈希表 + 双向链表
3. **拼写检查器**：哈希集合 + 编辑距离
4. **社交网络好友推荐**：图 + BFS

## 易错点
- 过早优化：不经分析就选择复杂结构
- 选型固定：不考虑组合使用多种结构
- 忽略常数因子：在小数据量上选 O(log n) 比 O(n) 更差
- 重复造轮子：Python 标准库（heapq, bisect, deque, Counter）已提供高效实现
