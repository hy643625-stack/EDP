# 实验：链表实现与性能对比

## 任务目标
从零实现单链表，并与 Python 内置 list 进行性能对比，验证理论复杂度。

## 输入输出
- 输入：不同规模的测试数据集合
- 输出：链表 vs 内置 list 在各操作下的运行时间对比表

## 实现步骤
1. 实现 `SinglyLinkedList` 类，包含：
   - `append(data)` — 尾插
   - `prepend(data)` — 头插
   - `insert_after(node, data)` — 指定节点后插入
   - `delete(data)` — 删除第一个匹配元素
   - `find(data)` — 查找元素返回节点或 None
   - `__len__` — 返回长度
2. 编写性能测试脚本，对比以下操作在两种结构上的耗时：
   - 尾部追加 n 个元素
   - 头部插入 n 个元素
   - 随机位置查找 1000 次
3. 测试规模：n = 100, 1000, 10000
4. 将结果整理为表格，分析是否与理论复杂度一致

## 提交物
1. `singly_linked_list.py` — 单链表实现
2. `performance_benchmark.py` — 对比测试脚本
3. 实验结果表格和分析文字

## 反思问题
1. 在 n=100 时，list 的头部插入可能比链表头插还快？为什么？
2. 如果链表尾插需要 O(n)，实际测试中 list 的 append 和链表的尾插差距有多大？为什么？
3. Python 内置 list 有哪些优化使得它的某些操作比"理论"更快？
