# 线性表：顺序表与链表

## 学习目标
- 能区分顺序表（数组）和链表的存储结构与适用场景
- 能实现单链表的增删查改操作
- 能分析顺序表和链表各操作的时间复杂度
- 能解释头指针、头结点、循环链表和双向链表的作用

## 知识地图
```
线性表
├── 顺序表（数组）
│   ├── 随机访问 O(1)
│   ├── 插入/删除 O(n)
│   └── 动态扩容（摊销 O(1)）
├── 链表
│   ├── 单链表
│   │   ├── 头插法 O(1) / 尾插法 O(n)
│   │   └── 按值查找 O(n) / 按位置查找 O(n)
│   ├── 双向链表
│   │   └── 前驱 O(1) / 删除已知节点 O(1)
│   └── 循环链表
│       └── 判断表尾：p.next == head
└── 选型决策
    ├── 随机访问多 → 顺序表
    ├── 频繁插入删除 → 链表
    └── 内存关键 → 顺序表（无指针开销）
```

## 核心讲解

### 顺序表
顺序表使用一段连续的内存存储元素。逻辑位置相邻的元素物理位置也相邻。

**优点**：O(1) 随机访问，缓存友好
**缺点**：插入/删除需移动后续元素 O(n)，容量不足时需扩容

Python 的 `list` 底层是动态数组。当容量不够时，会分配更大的空间并复制元素，扩容策略通常为 1.125~2 倍增长。

### 链表
链表中的每个节点存储数据和指向下一个节点的指针，节点在内存中可以任意分布。

**核心操作复杂度**：

| 操作 | 顺序表 | 单链表（已知位置）| 单链表（未知位置）|
|------|--------|-----------------|-----------------|
| 访问第 i 个 | O(1) | O(n) | O(n) |
| 头部插入 | O(n) | O(1) | O(1) |
| 尾部插入 | O(1)* | O(1)** | O(n) |
| 中间插入 | O(n) | O(1) | O(n) |

*摊销 O(1) **有尾指针时 O(1)

### 头结点技巧
在链表的第一个数据节点前加一个不存数据的"头结点"，可以使插入和删除操作的代码在处理首节点时与其他位置统一，无需特殊判断。

### 循环链表
最后一个节点的 next 指向头结点（或第一个节点），构成环路。适合需要循环遍历的场景，如约瑟夫问题。

## 典型例子
1. **反转单链表**：经典面试题，考察指针操作
2. **合并两个有序链表**：归并思想，O(n+m)
3. **判断链表是否有环**：快慢指针（Floyd 判圈算法）
4. **LRU 缓存**：双向链表 + 哈希表

## 易错点
- 链表中插入节点时，先连新节点再断旧链，否则会丢失后继
- 删除头节点时需更新头指针
- 遍历时忘记保存前驱节点，导致无法执行删除
- 忘记处理空链表边界情况
