# 学习智能体工作台设计说明

## 目标

在现有 EveryDayPerfect 项目上新增一个可独立使用的“学习智能体工作台”，用于承接以下能力：

- 对话式学习画像构建
- 多智能体协同资源生成
- 个性化学习路径规划
- AI 可配置增强，本地规则可兜底

本轮实现遵循“先最小可用、再逐步深化”的策略，不直接重做整个产品，也不强依赖数据库扩表。

## 当前实现

### 前端入口

- 新增应用页签：`Learning`
- 对应组件：
  - `frontend/src/features/learning/LearningStudioTab.tsx`
  - `frontend/src/features/learning/SimpleMarkdown.tsx`
- 本地持久化：
  - `frontend/src/lib/learningStudio.ts`

### 后端接口

- `GET /v1/learning/workbench`
  - 返回课程库、多智能体描述、画像维度、AI 运行状态
- `POST /v1/learning/profile`
  - 根据自然语言学习描述构建学习画像
- `POST /v1/learning/resource-package`
  - 生成资源包、学习路径、评估面板

对应代码：

- `backend/app/api/learning.py`
- `backend/app/services/learning_agent_service.py`

## 架构说明

### 1. 课程知识库

当前内置了 3 门课程：

- Python 程序设计
- 数据结构与算法
- 高等数学

每门课程都包含课程摘要、标签、模块列表、核心知识点和学习结果，用于作为资源生成与路径规划的基础知识骨架。

### 2. 学习画像

当前至少输出 8 个维度：

- 基础水平
- 目标导向
- 时间带宽
- 学习偏好
- 练习方式
- 节奏容忍度
- 风险因子
- 支持策略

同时补充：

- 优势信号
- 风险提醒
- 建议追问
- 聚焦模块

### 3. 多智能体协同

当前显式建模了 6 个智能体：

- 学习画像智能体
- 路径规划智能体
- 课程讲解智能体
- 练习设计智能体
- 资源策展智能体
- 督学教练智能体

后端会返回执行链路摘要，前端在工作台中可视化展示。

### 4. 资源生成

当前资源包默认输出 6 类资源：

- 主线讲解
- 课程脑图
- 分层练习包
- 精读与笔记指南
- 案例 / 实验任务
- 复盘清单

资源正文使用 Markdown 文本结构返回，前端用轻量渲染组件展示，并支持导出 Markdown。

### 5. AI 策略

当前延续项目原有 AI 设置体系：

- 未配置或关闭 AI：使用本地规则算法
- 已配置 AI：先生成本地资源骨架，再尝试模型增强
- AI 失败：自动回退本地规则

这保证了：

- 功能可用性不依赖云端
- 用户配置 AI 后可以获得更自然的内容
- 不会因为 AI 不通导致整页不可用

## 为什么本轮没有直接扩数据库

这是有意为之。

当前这一轮主要目标是把“学习智能体产品能力”先做成可展示、可测试、可继续迭代的独立模块。若一开始就把课程库、会话历史、学习档案、资源包版本全部入库，会显著增加迁移、打包、兼容与维护成本。

因此本轮采用：

- 后端无状态生成
- 前端本地持久化草稿
- AI 设置复用现有配置体系

后续如果要进入正式比赛版/交付版，可继续补：

- 学习会话历史表
- 课程库可配置化
- 用户级学习档案表
- 资源包版本记录
- 教学反馈闭环

## 后续建议

建议按下面顺序继续完善：

1. 增加“历史会话 / 历史资源包”数据库存储
2. 增加课程知识库导入能力（JSON / Markdown / 教材摘要）
3. 增加学习效果回评与错题归档
4. 为多智能体链路增加更细的执行状态与流式反馈
5. 如果面向比赛，再补“教师端 / 管理端 / 用户成长曲线”
