# 故障排查

## 1. 启动后前端显示“无法连接服务”
- 确认后端是否在运行：`http://127.0.0.1:18765/health`
- 若无响应，重新执行：`.\start_everydayperfect.ps1`
- 检查端口占用：`18765` 和 `5173`

## 2. 桌面版出现请求超时
- 先重启应用。
- 若仍异常，查看日志：`%LOCALAPPDATA%\EveryDayPerfect\logs\desktop-launcher.log`
- 优先使用最新安装包版本。

## 3. 任务创建无反应
- 打开浏览器开发者工具查看接口请求是否失败。
- 后端接口 `/v1/tasks` 返回 `success: true` 但页面不更新时，刷新页面并重试。

## 4. 脚本执行被禁止
在 PowerShell 执行：
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## 5. 仍无法定位
- 截图报错页。
- 复制诊断信息（应用内按钮）。
- 一并提供日志文件。

## 6. `git push` 连不上 GitHub
- 如果浏览器能上 GitHub，但 `git push` 提示 `Failed to connect to github.com port 443`，优先检查代理。
- 这通常说明浏览器走了系统代理，但 Git 没走代理。
- 处理方法见：
  `docs/GitHub发布与代理处理手册.md`

## 7. `gh` 已安装但 `auth status` 显示未登录
- 先确认 `gh.exe` 路径正确。
- 如果当前网络依赖本地代理，先补：

```powershell
$env:HTTP_PROXY='http://127.0.0.1:7897'
$env:HTTPS_PROXY='http://127.0.0.1:7897'
```

- 再执行：

```powershell
& 'C:\Program Files\GitHub CLI\gh.exe' auth status
```

- 完整处理见：
  `docs/GitHub发布与代理处理手册.md`
