📝 指示：彻底重构应用导航与心智模型 (The "Command Center" Pivot)
核心设计理念：
摒弃“先选任务，再看界面”的线性逻辑。任务 (Task) 不再是隔离的容器，而是属性的 标签 (Tag/Context)。未来的 AI Agent 需要一个全局视角，用户也一样。

重构动作清单：

1. 🔪 彻底砍掉顶部导航条的“任务下拉框”：

移除界面左上角的 [ 算法 v ] 下拉切换器。

用户不再需要“进入”某个任务。

2. 💥 拆除 Overview 页面：

从主导航 Tab (Overview, Records, Todos, Time, Stats) 中彻底删除 Overview。

原 Overview 页面里的“任务管理 (新建/删除任务)” 移动到一个独立的 ⚙️ 偏好设置 / 项目管理 侧边栏或全局弹窗中。这里不应该占用日常最高频的 C 位。

3. 🌟 将 Records 升级为默认的全局 Home (今日指挥中心)：

当用户打开 App，默认进入 Records (或改名为 Home / Focus)。

不再做数据隔离： 无论属于“算法”还是“读书”，所有今天需要处理的属性，全部平铺或按类别折叠展示在今日列表里。

视觉区分： 在每个属性卡片上，用彩色的小 Tag 来标识它属于哪个任务（例如：🔴 算法 | 🔵 读书）,若只想看部分任务，就拿标签筛选他们

这就是真正的“现代看板”：一眼看清今天所有的事，无需来回切换。

4. 🤖 引入全局事件收件箱 (AI Copilot Inbox)：

为了容纳“高优先级、跨任务”的全局事件（如：周期结束待结算、AI 洞察、目标偏离警告）。

在全新的 Records (Home) 页面最顶部，也就是日期选择器的上方，开辟一个专属的 Inbox / AI 洞察区

这个区域平时是折叠/隐藏的。一旦有跨任务的重大事件（比如前面的周期结算），它就会像“通知中心”一样展开，用极具情绪价值的 UI 提醒用户。

5. 📊 Stats 和 Time 页面的适配：

既然左上角的全局任务切换没了，怎么看单独“算法”的统计？

答案是：把过滤器下放到具体的页面。在 Stats 和 Time 页面内部提供一个 [筛选: 全局/算法/读书] 的筛选项。默认看全局，想深度分析再选具体的。
