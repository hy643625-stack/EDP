📝 指示：基于“覆盖逻辑”重构高级周期设置 UI
数据逻辑对齐：
后端已实现 target_overrides 逻辑，优先级为：组 (e.g. "1,2") > 单天 (e.g. "3") > daily > 默认值前端需通过“规则卡片”来收集这些数据

UI 重构动作：

基础设置区（兜底逻辑）：

保留 目标值 输入框，它对应最底层的基线 target_value

增加一个开关 [x] 开启高级周期覆盖，开启后显示下方的规则列表

规则列表区 (Rule List)：

允许用户通过 + 添加覆盖规则 按钮创建多张卡片

每张卡片代表一个 target_overrides 的条目：

天数选择： 提供周一至周日的 7 个按钮（多选）

模式选择： 一个二选一开关：[ 独立结算 / 共享目标 ]

目标值输入： 针对这几天设定的特定目标值

自动化 Key 生成逻辑（对用户透明）：

如果用户选了【周一、周二】+【共享目标】，前端自动将其处理为 Key "1,2"

如果用户选了【周三】+【独立结算】，前端处理为 Key "3"

如果用户设置了全局默认，处理为 "daily"

冲突处理与反馈（极简）：

互斥逻辑： 当一个星期值（如“周一”）已经被某个规则占用时，在其他规则卡片中该星期按钮应变为“禁用”或“置灰”状态，并提示“已在规则1中定义”这保证了 Key 的唯一性

实时摘要： 在列表下方显示一行小字预览，例如：“当前设定：周一二共享120，周三独立40，其余日子默认为60”
